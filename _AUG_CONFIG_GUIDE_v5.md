# AUG_CONFIG Guide (offline_aug_v5 기준)

이 문서는 `offline_aug_v5.py`의 `AUG_CONFIG`를 **그대로 기준**으로, 각 증강 기법이 이미지에 미치는 영향과 파라미터 범위(유효 범위), 값이 커지거나 작아질 때 생기는 **바리에이션 변화**를 상세히 정리합니다.

> 참고: 이 스크립트는 Albumentations를 사용합니다. 파라미터의 “가능 범위”는 Albumentations의 일반적인 제약(예: 픽셀 범위 0–255, 비율은 -1~1 등)과 실제 변환 의미를 기준으로 설명합니다.  
> Albumentations 버전 차이로 일부 파라미터의 상세 제약(예: 내부 클램핑/형변환)이 다를 수 있으니, 설치 버전에서 에러가 나면 해당 항목을 우선 점검하세요.

---

## 현재 v5의 기본 설정(요약)

아래가 v5에서 사용 중인 `AUG_CONFIG` 예시입니다.

```python
AUG_CONFIG = {
        # ---- Weather 계열 ----
        'Rain': {
            'enable': True,
            'brightness_coefficient': (0.85, 0.95),
            'drop_width': (1, 2),
            'blur_value': (3, 5),
        },
        'Shadow': {
            'enable': True,
            'num_shadows_lower': (1, 2),
            'num_shadows_upper': (1, 3),
            'shadow_dimension': (4, 6),
        },
        'Snow': {
            'enable': True,
            'brightness_coeff': (2.0, 3.0),
        },
        'RGBShift': {
            'enable': True,
            'r_shift_limit': (-20, 20),
            'g_shift_limit': (-20, 20),
            'b_shift_limit': (-20, 20),
        },
        'Solarize': {'enable': True},
        'Sepia': {'enable': False},
        'Blur': {
            'enable': True,
            'blur_limit': (3, 7),
        },

        # ---- Geo 계열 (ShiftScaleRotate는 아래 3개 enable 조합으로 구성) ----
        'Geo_Rotate': {'enable': True, 'rotate_limit': (-30, 30)},
        'Geo_Shift': {'enable': True, 'shift_limit': (-0.10, 0.10)},
        'Geo_Scale': {'enable': True, 'scale_limit': (-0.20, 0.20)},
    }
```

- `AUG_PER_CLASS=500` : **클래스당** 증강 이미지 생성량
- `WEATHER_RATIO=0.5` : Weather 계열(Rain/Shadow/Snow/RGBShift/Solarize/Sepia/Blur)과 Geo 계열의 생성 비율(대략적인 비중)
- `SEED=42` : 재현성(현실적 범위) — 동일 환경/동일 입력에서 seed가 같으면 결과가 최대한 동일하게 나오도록 설계

---

## 공통 규칙: (min, max) 튜플의 의미

`(min, max)` 형태의 값은 **증강 샘플 생성마다** 균등분포로 랜덤 샘플링됩니다.

- **범위를 좁히면**: 결과가 덜 다양해지고(변동 폭 감소) 원본과 비슷해짐
- **범위를 넓히면**: 바리에이션이 커지지만, 과도하면 “현실성 저하 / 학습 방해(도메인 시프트)”가 증가

---

## 1) Weather 계열(광학/환경 효과)

### 1.1 Rain (`A.RandomRain`)
비 오는 상황처럼 “빗줄기 + 약간의 흐림 + 밝기 변화”를 부여합니다.

**파라미터**
- `brightness_coefficient` (float, 보통 0~1 근처 권장)
  - 의미: 전반적인 밝기 계수(곱셈).  
  - **작아질수록** 이미지가 더 어두워짐 → 흐린 날/역광 느낌 강화  
  - **커질수록** 원본 밝기에 가까워짐(효과 약해짐)
  - 권장: `0.7~1.0` 범위 안에서 조절(너무 낮으면 정보 소실)
- `drop_width` (int, 양의 정수)
  - 의미: 빗줄기(드롭) “두께”
  - **작아질수록** 가늘고 촘촘한 느낌, 디테일 유지  
  - **커질수록** 굵은 빗줄기 → 가림/노이즈 증가(과하면 병징/엽맥이 사라짐)
  - 권장: `1~3` (대형 이미지를 강하게 흔들고 싶을 때만 더 크게)
- `blur_value` (int, 양의 정수; 보통 1 이상, 홀수 권장되는 경우가 많음)
  - 의미: 빗효과에 동반되는 블러 강도(또는 블러 커널 크기 성격)
  - **작아질수록** 빗줄기 선명, 원본 디테일 보존  
  - **커질수록** 흐림 증가 → 디테일 손실 증가(과하면 학습에 악영향)
  - 권장: `3~7` 정도에서 테스트

---

### 1.2 Shadow (`A.RandomShadow`)
부분 영역에 그림자를 만들어 “구름/차광/캐노피 그림자” 같은 광량 불균일을 모사합니다.

**파라미터**
- `num_shadows_lower` / `num_shadows_upper` (int, 양의 정수; lower ≤ upper)
  - 의미: 한 이미지에 생성되는 그림자 개수 범위
  - **개수가 늘수록** 다양한 위치/모양의 명암 변화가 생김  
  - 너무 많으면 식물 전체가 어둡게 덮여 “클래스 단서”가 약해질 수 있음
  - 권장: 실내/하우스면 `1~2`, 야외 강한 차광이면 `1~4`도 가능
- `shadow_dimension` (int, 양의 정수)
  - 의미: 그림자 다각형(마스크) 모양의 복잡도/크기 성격(구현에 따라 다각형 꼭짓점 수/스케일로 쓰임)
  - **작아질수록** 작은 그림자 조각 느낌, 국소 변화  
  - **커질수록** 큰 면적을 덮는 그림자 가능 → 전반 밝기 저하
  - 권장: `3~7` 사이에서 조절(너무 크면 거의 “야간”처럼 됨)

---

### 1.3 Snow (`A.RandomSnow`)
눈/서리/하얀 입자 느낌을 부여합니다. 작물 표면 반사/입자 노이즈를 모사할 때 사용됩니다.

**파라미터**
- `brightness_coeff` (float, 보통 0~5 사이에서 사용)
  - 의미: 눈 효과의 밝기/강도 계수 성격
  - **작아질수록** 약한 서리/미세한 흰 점  
  - **커질수록** 강한 눈, 하이라이트가 크게 증가 → 과하면 “하얀 덮임”처럼 됨
  - 권장: `0.5~3.5` 정도(너무 크면 병징 대비가 무너짐)

---

### 1.4 RGBShift (`A.RGBShift`)
RGB 채널을 각기 이동시켜 색감을 변형합니다. 촬영 장비/화이트밸런스 차이를 모사할 때 유용합니다.

**파라미터**
- `r_shift_limit`, `g_shift_limit`, `b_shift_limit` (int, 대체로 -255~255)
  - 의미: 채널별 픽셀 값 이동(더하기/빼기)
  - **절대값이 작을수록** 미세 색감 변화(현실적)  
  - **절대값이 클수록** 색 왜곡이 강함(도메인 시프트 증가)
  - 권장:
    - 현실적 장비 변동: `±5~±25`
    - 다양한 조명 조건까지 포함: `±10~±40`
    - `±60` 이상은 “특수효과”에 가까워질 수 있음

---

### 1.5 Solarize (`A.Solarize`)
밝은/어두운 영역을 특정 임계값 기준으로 반전하는 효과(필름/센서 특이 현상처럼 보일 수 있음).

- v5 설정은 파라미터 없이 `enable`만 있음 → Albumentations 기본값이 적용됩니다.
- 바리에이션을 더 통제하려면 보통 `threshold`(0~255) 같은 파라미터를 둡니다(버전/클래스에 따라 이름 다를 수 있음).

**일반적인 조절 원리(파라미터를 추가한다고 가정하면)**
- `threshold`가 **낮아지면** 더 많은 픽셀이 반전 → 효과 강함
- `threshold`가 **높아지면** 반전 영역이 줄어듦 → 효과 약함

> Solarize는 식물병징 진단에서 “과한 도메인 시프트”가 될 수 있어, 목적이 **센서/후처리 변동에 대한 강건성**이 아니라면 `enable=False`도 고려할 만합니다.

---

### 1.6 Sepia (`A.ToSepia`)
갈색/세피아 톤으로 전체 색조를 변형합니다.

- v5 기본은 `enable=False`
- 세피아는 자연 촬영 조건과 거리가 있어, **실제 사용 목적이 분명할 때만** 켜는 것을 권장합니다(예: 오래된 카메라/필터 환경을 일부러 포함).

---

### 1.7 Blur (`A.Blur`)
전체에 블러(흐림)를 적용하여 초점 흐림/움직임/카메라 흔들림 등을 모사합니다.

**파라미터**
- `blur_limit` (int 또는 (min,max); 일반적으로 **3 이상 홀수**가 안전)
  - 의미: 블러 커널 크기 범위
  - **작아질수록** 약한 흐림(현실적), 디테일 유지  
  - **커질수록** 흐림 강함, 디테일 손실 증가(병징/텍스처가 날아갈 수 있음)
  - 권장: `3~9` (고해상도에서만 더 크게 고려)

---

## 2) Geo 계열(기하학 변환)

v5는 Geo를 “Rotate / Shift / Scale” 3개로 나눠 enable 조합으로 `ShiftScaleRotate`를 구성합니다.
- `Geo_Rotate`만 켜면 회전만,
- `Geo_Shift`만 켜면 이동만,
- `Geo_Scale`만 켜면 스케일만,
- 여러 개가 켜져 있으면 합성 변환이 됩니다.

### 2.1 Geo_Rotate (`A.Rotate`)
**파라미터**
- `rotate_limit` (도 단위, 보통 -180~180 이내)
  - 의미: 회전 각도 범위
  - **범위가 작으면** 미세한 틸트만 발생(현실적)  
  - **범위가 커지면** 이미지가 크게 돌아가며, 가장자리 빈 영역(패딩/크롭)이 늘어날 수 있음
  - 권장: 작물 잎 데이터는 보통 `±10~±45`가 안정적

### 2.2 Geo_Shift (`A.ShiftScaleRotate`의 shift component)
**파라미터**
- `shift_limit` (비율, 보통 -1.0~1.0; 0.1이면 폭/높이의 10% 이동)
  - 의미: x/y 방향 평행 이동 범위
  - **작아질수록** 중앙 유지 → 객체가 프레임에 잘 남음  
  - **커질수록** 객체가 프레임 밖으로 나갈 위험 증가(특히 잎이 큰 경우)
  - 권장: `±0.05~±0.20`

### 2.3 Geo_Scale (`A.ShiftScaleRotate`의 scale component)
**파라미터**
- `scale_limit` (비율, 보통 -0.9~몇까지; 내부적으로 scale=1+u 형태인 경우 많음)
  - 의미: 확대/축소 범위
  - 예: `(-0.2, 0.2)`이면 scale이 대략 `[0.8, 1.2]` 범위로 변함
  - **절대값이 작으면** 미세한 줌 변화  
  - **절대값이 커지면** 크게 확대/축소되어 디테일/프레이밍이 크게 바뀜
  - 권장: `±0.05~±0.30`

## 3) Prefix(파일명 앞 번호) 규칙 — AUG_CONFIG 선언 순서 기반

v5는 **AUG_CONFIG에 선언된 순서대로** 0,1,2,… prefix를 부여합니다.

- 예: AUG_CONFIG가 `Rain → Shadow → Snow → RGBShift → Solarize → Sepia → Blur → Geo_Rotate → Geo_Shift → Geo_Scale` 순서라면
  - Rain: `0-...`
  - Shadow: `1-...`
  - ...
  - Blur: `6-...`
  - Geo 계열: 실제 파일 저장 시에는 통합 이름 `Geo_Transform`을 쓰되, prefix 번호는 “Geo_* 중 enable이 하나라도 켜진 첫 번째 Geo 항목의 위치”를 기준으로 맞춥니다.

> 이렇게 하면 “코드/가이드/데이터셋”에서 기법 순서가 항상 일치해 추적이 쉬워집니다.

---

## 4) 값 튜닝 가이드(실전 팁)

- **현실적 바리에이션**이 목적이면  
  - RGBShift: `±10~±25`  
  - Blur: `3~7`  
  - Rotate: `±15~±45`  
  - Shift: `±0.05~±0.15`  
  - Scale: `±0.05~±0.25`  
  - Rain/Shadow는 강도 과하면 병징 단서가 가려질 수 있으니 먼저 약하게 시작
- **강건성 극대화**가 목적이면(다양한 현장 환경 대응)  
  - Shadow 개수/면적(=dimension) 조금 확대  
  - Rain drop_width/blur_value 상향  
  - 단, 품질이 지나치게 떨어지면 “학습 방해(노이즈 학습)”로 역효과가 날 수 있어, 샘플 예시를 확인하면서 단계적으로 확대하는 것이 안전합니다.

---

## 5) 흔한 오류 체크리스트(설정 관련)

- `num_shadows_lower > num_shadows_upper` 같은 역전 설정
- Blur의 `blur_limit`가 짝수/너무 작아 내부에서 오류(또는 자동 보정) 가능
- shift/scale이 너무 커서 객체가 프레임 밖으로 나가 분류 단서가 약해짐
- RGBShift 절대값이 너무 커서 “색이 뒤집힌 듯”한 도메인 시프트 발생
